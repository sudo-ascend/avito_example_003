# Шаблон landing page для частного специалиста / ИП

Статичный адаптивный одностраничный сайт на `HTML + CSS + JS` без CMS, базы данных и авторизации. Шаблон подходит для быстрой адаптации под юриста, бухгалтера, репетитора, психолога, консультанта, мастера услуг или небольшую компанию.

## Структура проекта

```text
.
|-- index.html
|-- README.md
`-- assets
    |-- css
    |   `-- styles.css
    |-- img
    |   |-- favicon.svg
    |   |-- og-cover.svg
    |   `-- specialist-portrait.svg
    `-- js
        `-- main.js
```

## Как запустить

### Вариант 1. Просто открыть файл

Откройте `index.html` в браузере.

### Вариант 2. Через локальный сервер

Если нужен локальный адрес для тестов, из корня проекта можно запустить:

```bash
python -m http.server 5500
```

После этого откройте `http://localhost:5500`.

## Что и где менять

### 1. Тексты и структура блоков

Меняются в файле `index.html`.

Основные зоны:

- Шапка, имя специалиста, телефон, меню: верхняя часть `index.html`
- Первый экран / оффер / CTA: секция `#hero`
- О специалисте: секция `#about`
- Услуги: секция `#services`
- Этапы работы: секция `#process`
- Преимущества: секция `#advantages`
- Отзывы: секция `#reviews`
- FAQ: секция `#faq`
- Контакты и форма: секция `#contacts`
- Подвал и политика: нижняя часть `index.html`

В HTML уже оставлены комментарии `АДАПТАЦИЯ`, чтобы быстрее находить нужные места.

### 2. Контакты

Тоже меняются в `index.html`.

Что заменить:

- телефон: `tel:+79991234567` и видимый номер
- email: `hello@mironov-demo.ru`
- Telegram: `https://t.me/demo_expert`
- WhatsApp: `https://wa.me/79991234567`
- адрес и график работы
- данные в footer

### 3. Услуги

Меняются в блоке `#services` файла `index.html`.

У каждой карточки можно быстро поменять:

- название
- описание
- текст кнопки
- иконку `svg` внутри карточки

### 4. Отзывы

Меняются в секции `#reviews` файла `index.html`.

Для каждой карточки доступны:

- имя
- город
- текст
- рейтинг

### 5. FAQ

Меняется в секции `#faq` файла `index.html`.

Можно добавлять и удалять `.faq__item` без переделки скриптов.

### 6. Изображения

Файлы лежат в `assets/img/`.

По умолчанию используются:

- `specialist-portrait.svg` — главное изображение в hero и в блоке о специалисте
- `og-cover.svg` — картинка для Open Graph
- `favicon.svg` — иконка сайта

Если хотите поставить фото клиента:

1. Сохраните изображение в `assets/img/`
2. Замените путь `src` в `index.html`
3. При необходимости обновите `og:image` в `<head>`

### 7. Цвета, отступы, визуальный стиль

Меняются в `assets/css/styles.css`.

Главная точка настройки находится в блоке `:root`:

- `--color-bg`
- `--color-bg-soft`
- `--color-accent`
- `--color-accent-dark`
- `--color-warm`
- `--color-ink`
- `--color-muted`

Там же можно менять скругления, тени и ширину контейнера.

### 8. Форма заявки

UI и базовая валидация уже готовы.

Логика находится в `assets/js/main.js`.

Сейчас форма работает как демо:

- валидирует имя, телефон и сообщение
- показывает статус отправки
- не отправляет данные на реальный backend

Точка интеграции помечена комментарием `АДАПТАЦИЯ` в `assets/js/main.js`.

Туда можно подключить:

- Formspree
- свой backend
- отправку в Telegram-бота
- email-обработчик

## Быстрая адаптация под новую нишу

Чтобы превратить шаблон под нового клиента, обычно достаточно:

1. Поменять имя, оффер и контакты в `index.html`
2. Переписать 6 карточек услуг
3. Заменить отзывы и FAQ
4. Подставить фото или фирменную графику
5. Подкрутить цветовые переменные в `assets/css/styles.css`
6. Подключить обработчик формы в `assets/js/main.js`

## Рекомендация перед публикацией

Перед выгрузкой на хостинг желательно:

- заменить демо-контент на реальные данные клиента
- обновить политику конфиденциальности
- проверить ссылки на мессенджеры и телефон
- заменить `https://example.com` в JSON-LD на реальный домен
- при необходимости добавить реальные фотографии и кейсы
